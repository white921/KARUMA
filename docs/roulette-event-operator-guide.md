# 🎰 ヨーロピアンルーレット 企画者向け運営手順

## できる人

以下のいずれかのロールを持つ人が操作できます。

- 企画主
- 企画運営
- 技術統括

## 開催前の準備

各部のパネルが設置済みか確認してください。パネルがない・作り直したい場合は、**対象のパネル用チャンネルで** `/パネル` を実行します。

> `/パネル` を実行できるのは技術統括のみです。既存の同じ部のパネルは置き換わります。

## 基本の進行

1. `/賭け開始` を実行し、開催するフェーズを選ぶ
2. 参加者がパネルからベットする
3. 時間になったら `/賭け終了`
4. 出た数字を確認して `/結果` を実行する
5. 次のラウンドがあれば 1 に戻る
6. イベント終了時に `/ボーナス付与` を実行する

> **必ず `/結果` まで完了してから、次の `/賭け開始` を実行してください。** 未精算のラウンドがある間は次の受付を開始できません。

## コマンド一覧

### `/賭け開始`

ベット受付を開始します。`fase` は次のいずれかを選択します。

- `1st`：赤・黒・偶数・奇数
- `2nd`：第1部の内容 + ダズン（1〜12 / 13〜24 / 25〜36）
- `3rd`：第2部の内容 + ストレートアップ・スプリット

実行後、該当する部のパネルで「賭けを開始する」ボタンが使えるようになります。

### `/賭け終了`

現在のベット受付を締め切ります。締切後は新しいベットを受け付けません。

### `/結果`

出た数字を `number` に **0〜36の整数**で指定します。結果を確定し、配当を自動処理します。実行したチャンネルには結果と当選者の一覧が投稿されます。

> 0が出た場合は、すべてのベットがハズレです。

### `/ボーナス付与`

前回のボーナス付与以降にベットした参加者へ、**1人につき30,000krm** を配布します。

このコマンドを実行した時点がイベントの区切りです。次回は、それ以降にベットした人が新しい配布対象になります。日付や設定の変更は不要です。

> ボーナス配布後に次のイベントを始める場合は、そのまま `/賭け開始` を実行してください。

## 参加者への案内事項

- ベットは各ラウンドにつき1人1回です。確定後の変更・賭け直しはできません。
- 賭け金は1krm以上の整数です。
- 配当は賭け金込みで、通常ベットは2倍、ダズンは3倍、ストレートアップは36倍、スプリットは18倍です。

## 運営時の注意

- 管理コマンドにはカテゴリ制限はありませんが、`/結果` はその実行チャンネルに結果を投稿します。参加者に見せたいチャンネルで実行してください。
- ラウンド番号は各部ごとの通算です。イベントが変わっても第1ラウンドには戻りません。
- `/ボーナス付与` は再実行しても、前回配布対象へ重ねて配布されません。新しいベットがなければ配布人数は0人です。
